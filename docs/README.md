# 根脉 · 家族口述史

> 用声音留住一生，让故事跨越时间

---

## 项目简介

**根脉**是一款面向老年人的口述自传应用。老人通过语音与 AI 对话，讲述自己的人生故事；AI 自动将语音转录、提炼结构化信息、生成叙事文章，最终形成一部完整的个人口述自传。

本项目由南京大学 2025 级本科生独立开发，参加智能应用开发与创新大赛交互组。

---

## 产品目标

| 用户 | 核心目标 |
|---|---|
| 老年人（讲述者） | 只需开口说话，AI 帮你把故事整理成自传 |
| 子女（阅读者） | 随时翻阅父母的一生，听见他们年轻时的声音 |

**核心理念**：AI 不替代人讲故事，只负责降低门槛——转录、整理、润色，让每一位老人都能留下属于自己的口述历史。

## 核心功能

### 一、章节化口述自传

自传按人生阶段划分为 **11 个章节**，引导老人系统性地讲述自己的一生：

| 章节 | 主题 |
|---|---|
| 第 1 章 | 我是谁 |
| 第 2 章 | 我的来处 |
| 第 3 章 | 童年岁月 |
| 第 4 章 | 求学时代 |
| 第 5 章 | 工作与理想 |
| 第 6 章 | 爱情与婚姻 |
| 第 7 章 | 为人亲长 |
| 第 8 章 | 坎坷与选择 |
| 第 9 章 | 高光与遗憾 |
| 第 10 章 | 现在的日子 |
| 第 11 章 | 寄语 |

每个章节有预设引导问题，AI 根据老人的回答层层追问，形成完整的对话记录。

---

### 二、AI 采访（核心亮点）

**采访流程：**

```
选择章节 → 开始 AI 采访 → 语音回答 → FunASR 转录
→ 审查转录文字 → 保存 → AI 三层处理
```

**AI 三层处理：**

```
第一层：原始对话记录（语音 + 转录文字，永久留存）
第二层：结构化摘录（时间、地点、人物、核心事件）
第三层：叙事化润色（第一人称自述文章，有温度，有细节）
```

三层内容在故事详情页分 Tab 展示，用户可以单独编辑和重新生成任意一层。

---

### 三、故事详情页

每个故事包含三个维度的内容：

- **📖 故事**：AI 润色后的第一人称叙事文章，也可切换查看原始转录
- **💬 对话记录**：完整的问答对，每轮回答附带语音播放器
- **🏷️ 结构化信息**：时间范围、核心标签、涉及人物、核心事件

所有内容均支持手动编辑，润色版和结构化信息支持一键重新生成。

---

### 四、我的故事

以章节为单位展示所有故事：

- 11 个章节卡片，显示完成状态和故事数量
- 点击章节展开该章节的所有故事
- 每章可继续补充录入新故事

---

### 五、家庭成员

管理与自己相关的家族成员：

- 按关系分组（父母 / 兄弟姐妹 / 伴侣 / 子女 / 其他）
- 每位成员显示相关故事数量
- 点击成员查看所有提到该人的故事

---

### 六、我的足迹

可视化展示人生迁徙轨迹：

- 地图上以时间顺序连线，呈现一生走过的地方
- 标记点大小反映该地点的记忆数量
- 支持从故事中 AI 自动提取地名和坐标
- 同一地点多个事件合并展示，避免重复

## 本地运行

### 环境要求

- Python 3.10+（推荐 Anaconda）
- Node.js 18+

### 后端启动

```bash
cd backend

# 安装依赖
pip install fastapi uvicorn sqlalchemy python-multipart \
    funasr modelscope torch torchaudio openai python-dotenv

# 配置环境变量（新建 .env 文件）
ARK_API_KEY=你的火山引擎API Key
ARK_MODEL=你的豆包模型端点ID
ARK_BASE_URL=https://ark.cn-beijing.volces.com/api/v3

# 初始化历史事件数据（首次运行）
python init_historical_events.py

# 启动后端
uvicorn main:app --reload
```

后端运行在 `http://localhost:8000`

### 前端启动

```bash
cd genmmai-web

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端运行在 `http://localhost:5173`，浏览器打开即可使用。

### 首次使用

打开 `http://localhost:5173`，填写基本信息（姓名、出生年份、性别）完成初始化，即可开始录入故事。

## 差异化亮点

**1. 语音永久留存**

文字可以整理润色，但祖辈的声音是唯一的。每一段录音都保存在本地，故事详情页可以随时播放原声。

**2. 三维度交叉索引**

同一个故事从时间（历史时间轴）、空间（足迹地图）、章节（自传结构）三个维度都能找到，信息密度高但不混乱。

**3. AI 降低门槛，不替代人**

AI 做的是转录、摘要、标签提取、叙事润色这些整理工作，故事本身永远是人讲的。老人只需要说话，其余交给系统。

**4. 章节化引导，告别茫然**

11 个人生章节的预设问题，让老人每次打开都知道今天聊什么，AI 根据回答层层追问，形成完整的生命史访谈。

**5. 家国一体的历史语境**

故事时间轴左侧自动显示对应年代的重大历史事件，老人的个人记忆与家国历史并排呈现，体现"家族史融入国家史"的立意。

## 演示路径（答辩用，约5分钟）

1. 打开主界面，展示章节进度（第1章·我是谁）
2. 点击"开始 AI 采访"，进入采访界面
3. 展示预设引导问题，语音回答，等待转录
4. 审查转录文字，点击保存，等待 AI 三层生成
5. 进入故事详情页，依次展示三个 Tab：叙事文章 / 对话记录（播放语音）/ 结构化信息
6. 切换到"我的故事"，展示章节视图
7. 切换到"我的足迹"，展示地图上的迁徙轨迹

**演示核心卖点**：点击播放语音那一刻——这是这个人留下的声音，文字可以整理，声音无法复原。
