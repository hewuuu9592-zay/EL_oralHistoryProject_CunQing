# 存青·孤寡倾诉与口述史构建项目

> 让孤独被看见、声音被听见，让记忆留下痕迹

---

## 项目简介

**存青** 是一款面向孤寡、空巢及独居老年人的AI口述回忆录应用。

我们目睹一个静默而庞大的现实：在不愁温饱的时代，占我国人口五分之一的老年人群却面临着严重的精神上的孤独——无人说话，没人在意，没有出口。2025年统计数据显示，我国60岁以上独居老人达1.2亿、空巢老人突破1.8亿，而23.76%的老年人有不同程度的孤独感受，26.4%的老年人存在不同程度的抑郁症状。日复一日的沉默，正一点点抽走晚年生活的光与暖，也让那些独一无二的人生记忆随着老去而沉没。

**存青** 为老人带来一位永不疲倦的倾听者、采访者、记录者。老人随时开口，AI便耐心回应、层层追问，将每一次倾诉自动整理成章节化的个人自传。文字、结构化信息、原始录音的三重留存，让生命的故事终有归处，让老人 **不再无聊、孤单，不再被遗忘**。

## 产品目标


| 用户           | 核心痛点               | 存青带来的改变                                         |
| ------------ | ------------------ | ----------------------------------------------- |
| 孤寡/独居老人（倾诉者） | 无人说话，情绪无处安放，日子漫长无聊 | 随时打开软件就有人听我讲述自己的故事；AI像晚辈一样追问、共情；不知不觉中留下了自己的一生回忆 |
| 子女/家属（阅读者）   | 不了解父母的内心世界，不知如何陪伴  | 随时翻阅父母的口述自传，听见他们的声音，看见他们的喜怒哀乐，找到陪伴的方向           |
| 社会记忆         | 一代人的故事随生命逝去而湮灭     | 每一段倾诉都被结构化保存，形成可传承的家族口述史                        |


## **核心理念**：存青从不试图替代人与人之间的交互，只是承托那些无人回应的时刻。让老人拥有专属于自己的倾听者，它永远耐心、永远在场，懂得追问、也懂得珍藏。

## 核心功能

### 一、章节化口述自传

自传按人生阶段划分为 **11 个章节**，引导老人系统性地讲述自己的一生：


| 章节     | 主题    |
| ------ | ----- |
| 第 1 章  | 我是谁   |
| 第 2 章  | 我的来处  |
| 第 3 章  | 童年岁月  |
| 第 4 章  | 求学时代  |
| 第 5 章  | 工作与理想 |
| 第 6 章  | 爱情与婚姻 |
| 第 7 章  | 为人亲长  |
| 第 8 章  | 坎坷与选择 |
| 第 9 章  | 高光与遗憾 |
| 第 10 章 | 现在的日子 |
| 第 11 章 | 寄语    |


每个章节有预设引导问题，AI 根据老人的回答层层追问，形成完整的对话记录。

---

### 二、AI 采访（核心亮点）

**采访流程：**

```
选择章节 → 开始 AI 采访 → 语音回答 → FunASR 转录
→ 审查转录文字 → 保存 → AI 三层处理
```

**AI 三层处理：**

```
第一层：原始对话记录（语音 + 转录文字，永久留存）
第二层：结构化摘录（时间、地点、人物、核心事件）
第三层：叙事化润色（第一人称自述文章，有温度，有细节）
```

三层内容在故事详情页分 Tab 展示，用户可以单独编辑和重新生成任意一层。

---

### 三、故事详情页

每个故事包含三个维度的内容：

- **故事**：AI 润色后的第一人称叙事文章，也可切换查看原始转录
- **对话记录**：完整的问答对，每轮回答附带语音播放器
- **结构化信息**：时间范围、核心标签、涉及人物、核心事件

所有内容均支持手动编辑，润色版和结构化信息支持一键重新生成。

---

### 四、我的故事

以章节为单位展示所有故事：

- 11 个章节卡片，显示完成状态和故事数量
- 点击章节展开该章节的所有故事
- 每章可继续补充录入新故事

---

### 五、家庭成员

管理与自己相关的家族成员：

- 按关系分组（父母 / 兄弟姐妹 / 伴侣 / 子女 / 其他）
- 每位成员显示相关故事数量
- 点击成员查看所有提到该人的故事

---

### 六、我的足迹

可视化展示人生迁徙轨迹：

- 地图上以时间顺序连线，呈现一生走过的地方
- 标记点大小反映该地点的记忆数量
- 支持从故事中 AI 自动提取地名和坐标
- 同一地点多个事件合并展示，避免重复

## 本地运行

### 环境要求

- Python 3.10+
- Node.js 18+

### 后端启动

```bash
cd backend

# 安装依赖
pip install fastapi uvicorn sqlalchemy python-multipart \
    funasr modelscope torch torchaudio openai python-dotenv

# 配置环境变量（新建 .env 文件）
ARK_API_KEY=你的火山引擎API Key
ARK_BASE_URL=https://ark.cn-beijing.volces.com/api/v3

# 启动后端
uvicorn main:app --reload
```

后端运行在 `http://localhost:8000`

### 前端启动

```bash
cd cunqing-web

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端运行在 `http://localhost:5173`，浏览器打开即可使用。

### 首次使用

打开 `http://localhost:5173`，填写基本信息（姓名、出生年份、性别）完成初始化，即可开始录入故事。

## 差异化亮点

**1. 语音永久留存**

文字可以整理润色，但祖辈的声音是唯一的。每一段录音都保存在本地，故事详情页可以随时播放、下载原声。

**2. 三维度交叉索引**

同一个故事从人物、空间（足迹地图）、章节（自传结构）三个维度都能找到，信息密度高但不混乱。

**3. 人本思想**

AI 做的是转录、摘要、标签提取、叙事润色这些整理工作，故事本身永远是人讲的。老人只需要说话，其余交给系统。

**4. 章节化引导**

11 个人生章节的预设问题，让老人每次打开都知道今天聊什么，AI 根据回答层层追问，形成完整的生命史访谈。
