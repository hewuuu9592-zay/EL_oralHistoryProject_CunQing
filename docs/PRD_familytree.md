# 产品需求文档（PRD）

**产品名称**：根脉 · 家族口述史  
**版本**：v1.0  
**撰写日期**：2026年6月  
**作者**：南京大学 交互组参赛团队

---

## 一、电梯演讲

> 根脉是一款以家族为单位的口述历史 App——老人用语音讲述自己的故事，AI 自动整理成结构化记录，子孙后代通过可视化的家族树和故事时间轴，看见祖辈走过的一生。每一段语音，都是留给家族的礼物。

---

## 二、产品背景与目标

### 一句话问题陈述

> 每一位老人离开，都带走了一段无法复原的历史。

### 真实场景

- 爷爷在世时讲过很多故事，但从没有人认真记录下来
- 家族有族谱，但只有名字和生卒年，没有活生生的人
- 想问外婆年轻时的故事，却不知道从哪里开始问
- 父母离开后，才发现对他们的童年一无所知

### 产品目标

| 用户 | 核心目标 |
|---|---|
| 老年人（讲述者） | 用最简单的方式（说话）留下自己的故事 |
| 子女/晚辈（整理者） | 帮长辈录入故事，构建家族的记忆档案 |
| 家族成员（阅读者） | 通过直观的可视化界面，了解家族历史与人物 |

---

## 三、核心概念

### 三个核心实体

**人物**：家族树上的每一个节点，有头像、姓名、生卒年、简介。

**故事**：产品的核心实体，所有内容的最小单元。每个故事包含：
- 语音文件（原始录音，永久留存）
- 转录文字（AI 自动生成，可编辑）
- 时间标签（故事发生的年份或年代）
- 人物标签（故事涉及的家族成员）
- 主题标签（家乡 / 工作 / 爱情 / 战争 / 童年 / 其他）
- AI 提炼的摘要（一句话概括）

**关系**：连接两个人物的线，代表他们之间的关系类型（父子、夫妻、兄弟等）。关系线上沉淀了所有同时包含这两个人的故事。

### 核心设计原则

> 故事只存一份，但从三个维度都能找到它。

同一个故事——比如爷爷和奶奶的相识经历——会同时出现在：
- B1 爷爷的人际关系图（爷爷→奶奶的关系线上）
- B2 爷爷的个人时间轴（1965年节点上）
- B3 爷爷的主题故事集（"爱情"主题下）

## 四、功能详细说明

### 界面 A：族谱（家族空间）

---

#### A1 家族树可视化【P0】

**功能描述**  
以树状图展示家族成员及其血缘关系，是进入所有内容的入口。

**界面元素**
- 每个节点：圆形头像 + 姓名 + 生卒年（小字）
- 连线：代表血缘关系，线上可显示关系类型（父子/夫妻/兄弟）
- 支持缩放和拖拽，家族成员多时不拥挤
- 点击节点进入人物卡片（界面B）
- 长按节点可编辑人物信息或添加子节点

**添加人物**
- 右下角"+"按钮
- 填写姓名（必填）、头像（可选）、生卒年（可选）、简介（可选）
- 选择与已有人物的关系（父母/子女/配偶/兄弟姐妹）

---

#### A2 家族概览数据【P1】

**功能描述**  
族谱页顶部展示家族的整体数据，给用户成就感。

**展示内容**
- 家族成员人数
- 已收录故事数
- 跨越年代（最早故事年份 → 今天）
- 最近新增故事时间

---

### 界面 B：人物卡片

点击族谱上任意人物进入，顶部显示人物头像、姓名、生卒年、一句话简介，下方三个 Tab。

---

#### B1 人际关系图【P0】

**功能描述**  
以该人物为中心节点，展示与其有故事连接的其他家族成员，连线上沉淀了两人之间的所有故事。

**界面元素**
- 中心：当前人物头像（较大）
- 周围：有故事连接的其他人物节点（较小）
- 连线：线的粗细代表故事数量，线上标注关系类型
- 点击连线：展开侧滑面板，显示这两个人之间的故事时间轴

**连线故事面板**
- 标题：「爷爷 × 奶奶的故事」
- 时间轴排列，每条故事显示：年份、主题标签、AI摘要
- 点击单条故事进入故事详情页

---

#### B2 个人时间轴【P0】

**功能描述**  
以时间为轴，展示该人物所经历的所有故事，是这个人一生的纵向记录。

**界面元素**
- 纵向时间线，从出生年份延伸至今（或去世年份）
- 每个故事作为时间节点，显示：年份、主题标签图标、AI摘要
- 支持按主题过滤（全部 / 家乡 / 工作 / 爱情 / 童年…）
- 点击故事节点进入故事详情页
- 右下角"+"添加新故事（进入语音录入流程）

---

#### B3 主题故事集【P0】

**功能描述**  
以主题为维度，将该人物的故事重新分组，方便按兴趣浏览。

**主题分类**
| 主题 | 图标 | 说明 |
|---|---|---|
| 家乡记忆 | 🏡 | 出生地、故乡风貌、童年玩伴 |
| 工作岁月 | 💼 | 职业经历、工作故事 |
| 爱情婚姻 | ❤️ | 相识、恋爱、婚姻故事 |
| 历史亲历 | 📜 | 重大历史事件的亲身经历 |
| 家族传承 | 🌳 | 家训、传统、祖辈叮嘱 |
| 童年往事 | 🎈 | 童年玩耍、学校生活 |
| 其他 | 📖 | 不属于以上分类的故事 |

**界面元素**
- 主题卡片网格，每张卡片显示主题名、故事数量
- 点击主题进入该主题的故事列表
- 每条故事显示：年份、AI摘要、时长

---

### 界面 C：故事详情页

---

#### C1 故事详情【P0】

**功能描述**  
展示单个故事的完整内容，是内容消费的核心页面。

**界面元素**
- 顶部：年份 + 主题标签
- 音频播放器：播放原始语音，显示波形和进度
- 转录文字：AI 转录的完整文字，排版优雅
- 涉及人物：故事中提到的家族成员头像列表，点击可跳转
- 底部：编辑按钮（可修改文字和标签）

---

### 界面 D：故事录入流程

---

#### D1 语音录入【P0】

**功能描述**  
引导用户（老人或子女代录）完成一段故事的录制和标注。

**录入步骤**

第一步：AI 引导提问（可选）
- 系统根据该人物的已有信息，生成一个引导问题
- 例：「还没有关于工作的故事，要不要说说您当年是做什么工作的？」
- 用户可以选择回答这个问题，或自由讲述

第二步：录音
- 大圆形录音按钮，点击开始/停止
- 显示录音时长和音量波形
- 支持分段录制（一个故事可以录多段）

第三步：AI 处理（自动）
- 语音转文字（expo-speech-recognition）
- AI 提取：时间信息、涉及人物、推荐主题标签、生成一句话摘要
- 检测到新人物时提示：「您提到了"表叔"，要把他加入族谱吗？」

第四步：确认标注
- 用户确认或修改：转录文字、时间标签、人物标签、主题标签
- 确认后保存，故事自动出现在 B1/B2/B3 对应位置

---

#### D2 AI 引导提问库【P1】

**功能描述**  
AI 根据人物已有故事的空白，主动生成有趣的引导问题，帮助用户想起更多故事。

**问题示例**
- 「您在哪里出生的？小时候的家是什么样子？」
- 「您是怎么认识奶奶的？」
- 「您经历过文化大革命吗？那时候您在做什么？」
- 「您最骄傲的一件事是什么？」

## 五、数据结构

### persons 表
| 字段 | 类型 | 说明 |
|---|---|---|
| id | String | 唯一ID |
| family_id | String | 所属家族ID |
| name | String | 姓名 |
| avatar_url | String | 头像图片路径 |
| birth_year | Integer | 出生年份 |
| death_year | Integer | 去世年份（在世则为空） |
| bio | String | 一句话简介 |
| created_at | Timestamp | 创建时间 |

### relationships 表
| 字段 | 类型 | 说明 |
|---|---|---|
| id | String | 唯一ID |
| person_a_id | String | 人物A |
| person_b_id | String | 人物B |
| relation_type | String | father/mother/spouse/sibling/child/other |
| label | String | 自定义关系描述（可选） |

### stories 表
| 字段 | 类型 | 说明 |
|---|---|---|
| id | String | 唯一ID |
| family_id | String | 所属家族ID |
| audio_url | String | 语音文件路径（永久留存） |
| transcript | String | AI转录文字 |
| summary | String | AI生成的一句话摘要 |
| year | Integer | 故事发生年份 |
| decade | String | 年代（如"1960s"） |
| theme | String | 主题标签 |
| person_ids | JSON | 涉及的人物ID列表 |
| created_at | Timestamp | 录入时间 |
| created_by | String | 录入者 |

### story_persons 表（故事-人物关联）
| 字段 | 类型 | 说明 |
|---|---|---|
| story_id | String | 故事ID |
| person_id | String | 人物ID |
| is_protagonist | Boolean | 是否为故事主角 |

## 六、技术栈

| 层级 | 技术 | 说明 |

| 前端 | React Native + Expo | 跨平台，复用 React 基础 |

## 八、最小可演示版本（MVP）

**演示路径（5分钟）**

1. 打开 App，展示已有的家族树（提前录入示例数据）
2. 点击「爷爷」节点，进入人物卡片
3. 切换到 B2 个人时间轴，展示爷爷一生的故事节点
4. 点击某个故事节点，进入故事详情页，播放原始语音
5. 切换到 B3 主题故事集，点击「爱情婚姻」，看到爷爷奶奶的故事
6. 演示录入新故事：点击「+」，录音，AI自动转录+生成摘要+推荐标签
7. 保存后回到时间轴，新故事出现在对应年份

**这个演示的感染力**：  
评委看到真实的语音被转化成结构化的家族记忆，会想到自己从未记录下来的祖辈故事。

## 九、差异化亮点（答辩时重点说）

1. **语音永久留存**：文字可以由后人整理，但祖辈的声音是唯一的。语音文件的留存比文字更有情感价值。

2. **三维度交叉索引**：同一个故事从时间（B2）、关系（B1）、主题（B3）三个维度都能找到，信息密度高但不混乱。

3. **AI 不替代人，而是降低门槛**：AI 做的是转录、摘要、标签提取这些"整理"工作，故事本身永远是人讲的。这和市场上 AI 生成内容的方向相反。

4. **从个人记录到家族资产**：个人日记是私密的，这个产品的内容天然是家族共享的，有社交传播属性。
